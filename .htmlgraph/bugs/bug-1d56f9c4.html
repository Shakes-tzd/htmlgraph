link add on plan destroys interactive HTML — generic writeNode overwrites custom template

Todo Medium Priority

Caused By:

  • feat-2ad944a4

Part Of:

  • trk-f8fd5c66

Description

Running 'htmlgraph link add plan-xxx trk-xxx --rel implemented_in' calls writeNode() which uses the generic node template, overwriting the entire interactive plan HTML (dagre graph, section cards, approval UI, critique results) with a bare title/status/edges view. Any mutation (edge add, status change) on a plan node destroys its custom content. Root cause: writeNode() doesn't distinguish between plan HTML (custom template) and regular work item HTML (generic template). Fix: either skip writeNode for plan mutations and only update the specific HTML attribute, or detect plan type and preserve custom content.
