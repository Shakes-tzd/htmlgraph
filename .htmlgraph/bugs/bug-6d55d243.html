Description

When running git commit in a worktree (.claude/worktrees/trk-*), the pre-commit hook outputs 'Warning: no packages/go directory found, skipping Go checks.' This means Go quality gates (build, vet, test) are NOT running in pre-commit for worktree commits. The hook likely looks for a 'packages/go' directory that doesn't exist — the Go source is at cmd/ and internal/. The hook should detect Go projects via go.mod presence, not a packages/go directory.
