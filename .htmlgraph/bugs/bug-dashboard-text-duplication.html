Dashboard Activity Text Duplication

Done Medium Priority Bug

Related:

  • Feature: Self-Tracking

Affects:

  • Dashboard (index.html)

Description

Tool names were displayed twice in the dashboard activity log, appearing as garbled text like "mcp__pluginmcpyplugin..." when viewing session activity entries.

Root Cause

The li.textContent already included the tool name prefix, but the rendering code was prepending it again when displaying the summary.

Resolution

Fixed in index.html by stripping the tool prefix from the summary text if it's already present before displaying.

Fix Steps

  1. Identified duplicate tool name in activity log rendering
  2. Added prefix stripping logic in index.html
  3. Verified fix via Playwright screenshot
