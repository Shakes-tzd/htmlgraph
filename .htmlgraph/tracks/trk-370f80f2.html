Pre-Work Validation Hook with Auto-Spike Integration

Planned Critical Priority Created: 2025-12-25

Enforce HtmlGraph workflow by validating work items before code changes

Overview

Implement PreToolUse hook that enforces HtmlGraph workflow by requiring active work items for code changes. Integrates with auto-spike system and applies decision framework.

Context

Currently, the workflow is not enforced - agents can make code changes without creating work items first. This leads to attribution issues and violates the decision framework.

Requirements

⏳ Must not block auto-spike creation (session-init, transition)

must-have

⏳ Apply decision framework for mandatory work items (3+ files)

must-have

⏳ Use JSON permission decisions (deny/allow/ask)

must-have

⏳ Gracefully degrade if dependencies missing

should-have

  3. ⏳ Hook warns but allows single-file changes Warning test passes
  4. ⏳ get_active_work_item() returns correct Node or None Unit tests pass
  5. ⏳ All integration tests passing pytest exit code 0
