Multi-Agent Collaboration Specification

Draft Author: claude-code Created: 2025-12-21

Overview

HtmlGraph enables multiple AI agents (Claude Code, Codex CLI, Gemini CLI) to work together on the same project, with automatic session management, work attribution, and handoff capabilities. This track implements a comprehensive multi-agent collaboration system with feature claiming, handoff context, smart routing, work queue management, and enhanced session continuity.

Context

**Current Capabilities:** • Agent-Specific Sessions: Each agent maintains its own session with automatic tracking • Automatic Activity Attribution: Smart scoring based on file patterns, keywords, and priority • Session Continuity: Full context preservation across agent handoffs • Unified SDK: Same API for all agents **Gaps & Limitations:** 1. No Feature Assignment: No assigned_agent field, unclear ownership 2. No Work Claiming: Race conditions possible with multiple agents 3. Limited Handoff Context: Missing notes, recommendations, and blockers 4. No Agent Capabilities: Can't route work based on agent strengths 5. No Work Queue: Manual search for tasks, no priority visibility

Requirements

⏳ Add assigned_agent, claimed_at, claimed_by_session fields to Node model

⏳ Implement feature claim/release/auto-release CLI commands and SDK methods

⏳ Add handoff_notes, recommended_next, blockers fields to Session model

⏳ Create agent capabilities registry and smart routing algorithm

⏳ Implement work queue CLI and session-start hook integration

Acceptance Criteria

  1. ⏳ Multiple agents can work simultaneously without conflicts or duplicate work Start two features with different agents - no conflicts
  2. ⏳ Agent B has full context when continuing Agent A's work Agent A ends with notes, Agent B sees notes on session start
  3. ⏳ UI features automatically routed to browser-capable agents Feature with UI requirement only shown to Claude Code in work queue
  4. ⏳ Work queue shows available work with intelligent recommendations htmlgraph work queue shows prioritized, capability-matched features
  5. ⏳ Backward compatible - existing projects work without changes Existing HtmlGraph project continues functioning after upgrade
