NetworkX Graph Engine Integration

Planned High Priority Created: 2026-03-15

Integrate NetworkX as the graph algorithm engine for HtmlGraph. HTML files remain the source of truth (git-friendly). NetworkX provides battle-tested algorithms (shortest path, topological sort, cycle detection, PageRank, community detection) replacing ~500 lines of hand-rolled BFS/DFS in graph.py. The EdgeIndex feeds a nx.DiGraph that powers all relational queries.

Overview

Replace custom graph algorithms with NetworkX DiGraph, keeping HTML as the storage layer

Context

HtmlGraph's core philosophy is 'HTML is all you need' — HTML files are graph nodes, hyperlinks are edges, CSS selectors are the query language. The current graph.py has ~500 lines of hand-rolled BFS/DFS for shortest_path, transitive_deps, topological_sort, find_cycles, and find_bottlenecks. These work but lack the breadth and battle-testing of NetworkX. Meanwhile, the graph_edges SQLite table is defined but never populated, Collections don't support relational queries, and the CLI has no graph query commands.
