Browser-Native Query Interface

Planned High Priority Created: 2026-01-13

Inspired by Vercel's agent-browser project, this track adds browser-native query capabilities to HtmlGraph. Key innovations include accessibility-tree-based refs (@f1, @t1), semantic query DSL, AI-readable snapshots, and optional browser automation integration. The browser becomes a true graph database interface.

Overview

HtmlGraph stores data as HTML files with hyperlinks as edges. A browser can become more than just a viewer—it can be a query interface. Agent-browser's key insight is using accessibility trees instead of brittle DOM selectors. This track adopts that pattern: short stable refs, semantic queries, and snapshot-based AI reasoning, while keeping browser integration optional.

Context

Current state: HtmlGraph provides SDK and CLI for querying the graph, but uses long UUIDs and requires direct file access. Agent-browser demonstrates a CLI-first, ref-based approach that's AI-friendly. We can adopt these patterns without requiring agent-browser as a dependency.
