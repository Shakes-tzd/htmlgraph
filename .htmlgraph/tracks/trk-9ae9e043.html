Overview

Comprehensive optimization of HtmlGraph hooks system to improve performance, reduce code duplication, and enhance maintainability. Current State Issues: - SessionStart takes 2-7 seconds (sequential operations) - 189 lines of duplicated bootstrap code across 7 hooks - 3-5 database queries per PostToolUse event - 1,562 lines of unused/deprecated hooks - Pattern matching uses O(n*m) complexity instead of O(n) Target Outcomes: - SessionStart latency: 2-7s → 1-2s (50-60% improvement) - Query efficiency: 3-5 queries → 1-2 queries per event (60-75% reduction) - Code deduplication: Remove all 189 duplicate lines - Unused code removal: Clean up 1,562 unused lines - Pattern performance: O(n*m) → O(n) with compiled regex This track enables parallel execution across 4 independent phases: - Phase 1: Quick wins (cleanup, consolidation) - Phase 2: Performance (parallelization, batching, caching) - Phase 3: Architecture (refactoring, consolidation) - Phase 4: Polish (documentation, testing, integration)
