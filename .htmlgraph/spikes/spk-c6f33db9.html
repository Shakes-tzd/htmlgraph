## Correct SDK Agent Naming **RULE: Use actual model names, not arbitrary role names** ### Correct Pattern ```python from htmlgraph import SDK # Correct - use model name sdk = SDK(agent='claude-haiku-4-5-20251001') sdk = SDK(agent='claude-sonnet-4-5-20251105') sdk = SDK(agent='claude-opus-4-5-20251101') sdk = SDK(agent='gpt-4') sdk = SDK(agent='gemini-2.0-flash-exp') ``` ### Incorrect Pattern ```python # WRONG - arbitrary role names sdk = SDK(agent='coder') sdk = SDK(agent='status-updater') sdk = SDK(agent='cleanup') sdk = SDK(agent='explorer') ``` ### Why This Matters - HtmlGraph tracks which model/agent created features - Analytics queries filter by agent type - Session attribution requires correct model names - Role names don't map to actual agents ### When Delegating to Subagents In Task() delegations, include these instructions: ```python from htmlgraph import SDK # Get your actual model name (e.g., from environment or model config) sdk = SDK(agent='claude-opus-4-5-20251101') # Your actual model # For subagents spawned via Task() # The subagent should use its own model name sdk = SDK(agent='gpt-4') # If Codex spawner sdk = SDK(agent='gemini-2.0-flash-exp') # If Gemini spawner sdk = SDK(agent='claude-sonnet-4-5-20251105') # If Copilot spawner (Claude-based) ``` ### Model Name Reference | Spawner | Model Name | |---------|-----------| | Codex Spawner | gpt-4 | | Gemini Spawner | gemini-2.0-flash-exp | | Copilot Spawner | claude-sonnet-4-5-20251105 | | Claude (orchestrator) | claude-haiku-4-5-20251001, claude-sonnet-4-5-20251105, claude-opus-4-5-20251101 | ### Impact This bug affected: - Feature attribution (features attributed to 'coder' instead of actual model) - Analytics (queries by agent don't match reality) - Session tracking (agent name mismatches)
