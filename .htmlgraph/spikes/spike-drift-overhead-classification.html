Objective

Investigate and establish a classification system to distinguish between legitimate system overhead activities and true off-topic drift in activity tracking.

Context

Current drift-queue.json classification assigns high drift scores (0.71) to activities that are actually system overhead:

  • htmlgraph-tracker skill invocations (system tracking mechanism)
  • Reads from .htmlgraph/ internal metadata directory
  • Glob operations on .htmlgraph/bugs/ for file discovery

These are legitimate self-referential activities from HtmlGraph's self-tracking feature, not actual context drift.

Research Questions

  1. What activity types should be classified as "system overhead" vs "drift"?
  2. Should overhead activities be excluded entirely or weighted differently?
  3. How can we distinguish between feature-relevant operations and true divergence?
  4. What metrics best characterize true drift vs normal system activity?
  5. Should there be separate categories for: overhead, drift, related, and on-topic?

Approach

  1. Analyze current high-drift activities in drift-queue.json
  2. Categorize by type (skill, read, glob, etc.) and purpose
  3. Identify patterns that indicate system overhead vs genuine off-topic work
  4. Document classification rules and thresholds
  5. Propose updated drift calculation algorithm
  6. Design filtering/exclusion rules for drift calculation

Success Criteria

  • Clear definition of "system overhead" vs "drift" established
  • Classification rules documented in drift-queue.json
  • Current false-positive activities reclassified as overhead or filtered out
  • Remaining drift activities are genuinely off-topic
  • Bug #bug-high-drift-skill-invocations can be closed
