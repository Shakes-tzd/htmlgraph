## Root Cause The Activity Feed was showing `"UserQuery: {'prompt': '..."` instead of just the prompt text in the `input_summary` field for UserQuery events in SQLite. ## Investigation 1. **Database Query**: Found 72 UserQuery events with `evt-` prefix (generated by `track-event.py`) 2. **Code Analysis**: The `format_tool_summary()` function in `track-event.py` has logic to handle UserQuery (lines 580-587), but it was not being executed 3. **Root Issue**: The function was falling through to the else clause (line 604), which formats as `"{tool_name}: {str(tool_input)[:50]}"` ## Why It Happened Either: - `tool_name` parameter had whitespace or was not exactly "UserQuery" - An exception was silently caught (though no explicit exception handler existed) ## Fix Applied **File**: `packages/claude-plugin/.claude-plugin/hooks/scripts/track-event.py` **Changes**: 1. **Line 533**: Added normalization to strip whitespace from tool_name: `tool_name = (tool_name or "").strip()` 2. **Lines 580-590**: Added exception handling in UserQuery case with try/except 3. **Lines 593-602**: Added defensive fallback in the else clause that checks if `tool_input` contains `"prompt"` key and extracts just the prompt text ## Result - Future UserQuery events will show just the prompt text, not the dict representation - Resilient to edge cases where tool_name has whitespace or tool_input is malformed - Backward compatible - doesn't break existing functionality ## Verification - ✓ Ruff linting passed - ✓ Hook-related tests passed (60 passed, 1 skipped) - ✓ Event tests passed - ✓ No regressions in existing code
