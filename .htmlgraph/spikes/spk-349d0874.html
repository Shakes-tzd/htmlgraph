# Root Cause Analysis ## Problem The Activity Feed query in main.py was using `agent_id` column which is always "claude-code" (the orchestrator). The `subagent_type` column IS correctly populated for Task delegations but wasn't being used in the display. ## Solution Implemented Modified `/Users/shakes/DevProjects/htmlgraph/src/python/htmlgraph/api/main.py`: ### Query Change (line ~360) **Before:** ```sql SELECT e.event_id, e.agent_id, e.event_type, ... FROM agent_events e ``` **After:** ```sql SELECT e.event_id, COALESCE(e.subagent_type, e.agent_id) as agent_id, e.event_type, e.timestamp, e.tool_name, e.input_summary, e.output_summary, e.session_id, e.status, e.parent_event_id, e.cost_tokens, e.execution_duration_seconds, e.subagent_type FROM agent_events e ``` ### What COALESCE Does - Prefers `subagent_type` when available (Task delegations) - Falls back to `agent_id` when `subagent_type` is NULL (direct orchestrator actions) - No schema changes needed - just better use of existing data ## Expected Behavior After Fix - Task delegations show actual subagent names: - "gemini-2.0-flash-exp" for Gemini delegations - "gpt-4" for Codex delegations - "copilot" for Copilot delegations - Direct orchestrator actions still show "claude-code" - Proper agent attribution in Activity Feed and Agents panel ## Testing Steps 1. Restart the dashboard server: `uv run htmlgraph serve` 2. Open Activity Feed 3. Look for Task delegation events 4. Verify they show subagent names, not "claude-code" ## Files Modified - `src/python/htmlgraph/api/main.py` (lines 360-366, 416-433) ## Credit Root cause identified by Gemini investigation, implemented by Claude.
