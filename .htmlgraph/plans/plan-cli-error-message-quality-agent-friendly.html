Improve CLI error messages with recovery hints and agent-friendly guidance across 33 findings.

A. Design DiscussionPending

Current State

CLI error messages say what went wrong but not WHY or HOW TO FIX. Example: work item "b743beb4" not found — no hint about valid ID formats, no recovery command. AI agents hit these and enter retry loops.

Desired End State

Every error follows the pattern: what went wrong — what was expected. Run 'command' to fix. Shared helpers ensure consistency. The codebase already has good examples to follow (plugin.go:28, claude.go:223, tdd.go:70).

Audit Summary

33 findings across 4 severity levels: 2 critical (misleading exit codes), 18 high (zero guidance), 10 medium (missing context), 3 low (minor improvements). Three shared-fix patterns cover 70% of findings.

Open Questions

Error message length?

Longer messages give agents more guidance but add tokens to every error path. One-line follows the pattern: what went wrong — hint.

Helper location?

The shared error helpers (ErrNotFound etc.) need a home. Existing package is simpler; new package is more modular.

Should wip reset without --force exit 0 or 1?

Currently returns exit 1 which causes agents to enter retry loops. Exit 0 is friendlier but some CI scripts may rely on the non-zero exit to prevent accidental resets.

B. Structure OutlinePending

Shared Error Helpers

// internal/workitem/errors.go

// ErrNotFound returns a formatted error with ID format hints and list command.
// Example: work item "b743beb4" not found — IDs look like feat-*, bug-*, trk-*.
//          Run 'htmlgraph bug list' to see valid IDs.
func ErrNotFound(kind, id string) error

// ErrNotFoundOnDisk returns a formatted error when the HTML file is missing.
// Suggests running reindex to rebuild the index.
func ErrNotFoundOnDisk(kind, id string) error

// ErrNoActive returns a formatted error when no active items exist.
// Example: no active sessions — run 'htmlgraph session start' to begin.
func ErrNoActive(kind, recoveryCmd string) error

// ErrUnknownValue returns a formatted error listing valid options.
// Example: unknown type "foo" — valid types: feature, bug, spike, track
func ErrUnknownValue(field, value string, valid []string) error

Call Site Pattern

Each call site replaces its bare fmt.Errorf("X not found") with the appropriate helper, passing the kind (feature, bug, track) and the recovery command.
