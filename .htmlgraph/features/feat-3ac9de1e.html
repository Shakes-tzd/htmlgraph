Implementation Steps

  2. ✅ Add --description flag to all create commands. For bugs: also add --files flag (comma-separated affected file paths). Validate required fields on create — warn if missing, don't block
  3. ✅ Add htmlgraph check incomplete command that scans all work items and reports those missing required fields (e.g., bugs without description, features without steps)
  4. ✅ Auto-link git commits to active work item: in PostToolUse hook, when a git commit is detected (already captured in git_commits table), create an implemented_in edge from active feature/bug to the commit hash
  5. ✅ Parse work item ID from commit messages (e.g., 'fix: title (bug-xxx)' or 'feat: title (feat-xxx)') and auto-link even if no active work item is set — store as edge in HTML + graph_edges
  6. ✅ Extend htmlgraph feature show / bug show to display linked commits: query graph_edges for implemented_in edges where target starts with commit hash pattern, show hash + message + date
  7. ✅ Store affected_files as structured data in bug HTML (data-affected-files attribute or <ul data-files> section). Display in bug show output.
  8. ✅ Write tests: bug create without description → warning. feature create without steps → warning. git commit with (feat-xxx) in message → edge created. htmlgraph check incomplete → reports missing fields.
