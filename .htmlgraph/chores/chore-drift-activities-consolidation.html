  • File Operations: Create/read/list operations in .htmlgraph/ directory

Root Cause Hypothesis:

The drift classification algorithm may be over-weighting "infrastructure self-referential" operations. When the tracking system creates work items to consolidate its own activities, this creates a logical loop:

  1. High-drift activity detected (e.g., repeated Skill invocation)
  2. Tracking system creates work item to classify the drift
  3. Creating work item triggers file I/O (Read, Write, Bash)
  4. File I/O on .htmlgraph/ is classified as high-drift
  5. Loop continues - drift about drift about drift...

  • Workflow overhead - time spent managing drift classification vs. development
  • Context pollution - legitimate work items mixed with meta-tracking items
  • Feedback loop reduces signal-to-noise ratio over time
