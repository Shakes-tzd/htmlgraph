Description

The drift-queue.json is accumulating meta-activities (repeated Skill invocations, drift-queue reads) that are system infrastructure work, not feature development.

Problem:

  • Activities related to the tracking system itself (Skill invocations, drift-queue reads) are being attributed to feature-self-tracking
  • High drift scores (0.71) indicate these activities don't match the feature's normal patterns
  • No automated way to distinguish between "working on feature X" vs "system maintenance for feature tracking"

Solution:

  • Identify meta-activity patterns (tool = Skill with htmlgraph:htmlgraph-tracker, files matching .htmlgraph/*)
  • Classify these as infrastructure work with a separate work item
  • Keep feature tracking clean by excluding system activities from drift calculation

Examples of High-Drift Activities Detected:

  • Skill invocation: htmlgraph:htmlgraph-tracker (5 occurrences)
  • Read: drift-queue.json (2 occurrences)
  • All with drift_score ≥ 0.71
